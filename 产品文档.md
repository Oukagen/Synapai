# Synapai 突触简报 - 产品文档

## 1. 项目概述

**项目名称**：Synapai（突触简报）
**项目类型**：AI行业资讯聚合网站 + 内容管理系统
**核心功能**：聚合全球AI领域资讯，支持手动/自动文章管理，生成封面图，RSS订阅
**目标用户**：AI从业者、科技爱好者、内容编辑

### 技术栈
- **框架**：Next.js 15.1.0（App Router）
- **语言**：TypeScript
- **样式**：Tailwind CSS（但不主要使用className，而是CSS变量系统）
- **UI库**：Lucide React（图标）
- **内容解析**：gray-matter（Markdown frontmatter）、remark/remark-html（Markdown转HTML）
- **部署**：Vercel + GitHub集成

## 5. API接口

### 5.1 文章列表 POST `/api/posts`

**请求体**：
```typescript
{
  title: string;      // 必填
  date: string;       // YYYY-MM-DD格式，默认为当天
  category: string;    // 分类ID
  source_url: string;  // 来源URL
  description: string; // 文章描述
  cover_image: string; // 封面图URL
  is_featured: boolean;
  content: string;    // Markdown内容，必填
}
```

**处理逻辑**：
1. 验证title和content必填
2. 生成slug：从title生成，处理中文字符，截断50字符
3. 写入文件：`content/posts/{slug}.md`，格式：
   ```yaml
   ---
   title: "{title}"
   date: "{date}"
   category: "{category}"
   source_url: "{source_url}"
   description: "{description}"
   cover_image: "{cover_image}"
   is_featured: {true/false}
   ---
   {content}
   ```
4. 转义双引号（特别是中文引号""）防止YAML解析错误
5. 调用generatePostsJson()更新posts.json

**返回**：`{ success: true, slug }` 或 `{ error: "...", status: 500 }`

---

### 5.2 获取单篇文章 GET `/api/posts/[slug]`

**参数**：slug（URL参数，decodeURIComponent处理）

**处理逻辑**：
1. 查找`content/posts/{slug}.md`
2. 使用gray-matter解析frontmatter和content
3. 返回完整文章数据

**返回**：
```typescript
{
  slug: string;
  title: string;
  date: string;
  category: string;
  source_url: string;
  description: string;
  is_featured: boolean;
  cover_image: string;
  content: string;  // Markdown原文
}
```

**错误**：404时返回 `{ error: "文章不存在", status: 404 }`

---

### 5.3 更新文章 PUT `/api/posts/[slug]`

**参数**：slug（URL参数）

**请求体**：同POST `/api/posts`

**处理逻辑**：
1. 验证title和content必填
2. 检查文件存在
3. 覆盖写入frontmatter
4. 重新生成posts.json

---

### 5.4 删除文章 DELETE `/api/posts/[slug]`

**参数**：slug（URL参数）

**处理逻辑**：
1. 将文件移动到`content/trash/{slug}.md`（而非直接删除）
2. 重新生成posts.json

**返回**：`{ success: true }`

---

### 5.5 回收站 GET `/api/trash`

**返回**：`Post[]` - trash目录所有.md文件的元数据数组

---

### 5.6 回收站操作 POST `/api/trash`

**请求体**：`{ action: "restore" | "delete", slug: string }`

**restore逻辑**：
1. 从trash移动回posts目录
2. 重新生成posts.json

**delete逻辑**：
1. 永久删除文件（unlinkSync）

---

### 5.7 RSS订阅 GET `/feed`

**返回**：RSS 2.0 XML格式

**内容**：
- channel: Synapai - 突触简报
- 每篇文章的item：title、link、guid、pubDate、description、content:encoded
- language: zh-cn

---

### 5.8 自动更新 POST `/api/automation`

**请求体**：
```typescript
{
  llmConfig: {
    provider: string;
    apiKey: string;
    endpoint: string;
    model: string;
  };
  dataSources: DataSource[];  // 启用的数据源
  defaultCoverImage?: string;
}
```

**处理流程**：
1. **RSS抓取**：fetchRSS()遍历每个数据源的URL
   - 支持RSS和Atom格式
   - 使用正则`<entry[^>]*>([\s\S]*?)<\/entry>`或`<item[^>]*>([\s\S]*?)<\/item>`
   - 提取title、link、description（支持CDATA）、pubDate、imageUrl
   - 图片优先从media:thumbnail/media:content提取，其次从HTML description中提取img标签

2. **去重检查**：
   - 通过source_url检查文章是否已存在（articleExistsByUrl）
   - 也检查slug（generateSlug生成的）是否存在

3. **内容转换**：
   - 优先使用Jina AI（`https://r.jina.ai/{url}`）将原文转为Markdown
   - Jina AI失败时，使用RSS原始description作为fallback

4. **LLM生成**：
   - 调用配置的LLM API生成标题、摘要、分类、正文
   - 使用promptTemplate，替换`{content}`为抓取的内容
   - 解析LLM输出：按"标题：""摘要：""分类：""正文："分段

5. **分类映射**：
   ```
   大厂动态 -> tech-giants
   行业脉搏 -> industry-pulse
   模型追踪 -> model-tracking
   工具开箱 -> tool-unboxing
   热门skill -> hot-tools
   ```

6. **封面图生成**：
   - SVG生成，1200x630尺寸
   - 多色渐变背景（6组预设，基于slug hash选择）
   - frosted glass效果（多个白色椭圆+高斯模糊）
   - 关键词文字居中显示
   - 保存到`public/generated-covers/{slug}.svg`

7. **保存文章**：
   - 调用saveArticle()写入content/posts/{slug}.md
   - 转义中文引号防止YAML解析错误

8. **返回结果**：
   ```typescript
   {
     results: Array<{
       source: string;
       title: string;
       status: "success" | "skipped" | "error";
       error?: string;
     }>
   }
   ```

### 7.1 Slug生成规则

function generateSlug(title: string): string {
  return title
    .toLowerCase()
    .replace(/[^\w\s一-龥]/g, "")  // 只保留字母数字中文
    .replace(/\s+/g, "-")          // 空格转-
    .slice(0, 50) || "untitled-" + Date.now();  // 截断50字符
}

## 11. 错误处理

### 11.1 构建错误

**YAMLException: multiline key**
- 原因：frontmatter中的中文引号""未转义
- 解决：escapeQuotes函数转义所有双引号

**TypeScript implicit any**
- 原因：filter/map/sort/find回调参数未声明类型
- 解决：显式声明类型，如 `(f: string) => f.endsWith(".md")`

### 11.2 运行时错误

- Jina AI访问失败：fallback到RSS description
- LLM API错误：捕获异常，返回详细错误信息
- 文件不存在：返回404或空数组
